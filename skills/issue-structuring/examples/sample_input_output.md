# 論点整理スキル｜入出力サンプル

## サンプル入力

```text
来週、新規AI案件の方針会議。
クライアントは「まずPoC」と言っているが、現場はデータが無い。
経営はROIを欲しがっている。私は要件がふわっとして不安。
整理して。
```

## サンプル出力（理想イメージ）

## 0. 状況の言い換え（3行以内）
- 来週の方針会議で、AI案件の進め方を決める必要がある
- 現場はデータ不足、経営はROI、現場 pen はまずPoCと立場が違う
- 論点が「PoCをやるか」に矮小化され、本来決めるべきことが曖昧

## 1. Decision Ask
- 来週時点で、本案件を「PoC先行」で進めるか、「データ/業務前提の整備先行」で進めるか、どちらを公式方針にするか

## 2. 論点ツリー
### A. いま解くべき経営課題は何か？
- A1. コスト削減 / 売上 / リスク / 学習 のどれが主目的か
- A2. 3ヶ月後に「成功」と呼ぶ状態は何か

### B. PoCは何を検証する手段か？
- B1. 検証仮説は何か（何が分かればGo/NoGoか）
- B2. データが無い状態でPoCは成立するか

### C. データ・業務前提の現状は十分か？
- C1. 必要データの所在・品質・権限
- C2. 現場オペで変わる作業は何か

### D. 意思決定の進め方はどうするか？
- D1. 来週決めること / 再来週以降でよいこと
- D2. 経営・現場・情シスの合意条件

## 3. 今やる論点 Top3
1. A2 成功状態の定義 — 理由: これ無しにPoCもROIも設計不能
2. B1 検証仮説 — 理由: PoCが目的化することを防ぐ
3. C1 データ前提 — 理由: 現場の「データが無い」が致命傷か確認する

## 4. 今やらない論点
- 詳細モデル選定 — 理由: 検証仮説が先
- 年間の全社AIロードマップ — 理由: 本会議のスコープ外

## 5. 足りない事実
- 経営が欲しいROIの定義（金額/期間）
- 利用想定データ1つ分の有無
- 現場の協力時間（週何時間か）

## 6. 次アクション
| 誰 | 何 | いつまで |
|---|---|---|
| 自分 | 成功状態の案を3行で作成 | 会議2日前 |
| 自分 | 現場にデータ有無を1点確認 | 会議2日前 |
| 自分 | PoCで検証する仮説を1文化 | 会議前日 |

## 7. 次にAIへ投げるプロンプト
```text
上記のDecision AskとTop3論点を前提に、
来週の方針会議用に「比較表（PoC先行 vs 前提整備先行）」を作って。
各案について: 必要前提、1ヶ月の動き、リスク、決裁者への一言、を表で。
一般論禁止。この案件の言葉で。
```

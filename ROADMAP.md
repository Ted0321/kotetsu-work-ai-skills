# X公開ロードマップ

## 1. find-skills-ja — 2026-07-21

エージェント型の第1弾。「find-skillsは日本語で聞くと0件」という実測の問題提起とセットで公開し、リポジトリへの導線を作る。

確認する反応:

- 導入コマンド（`npx skills add`）が実行されるか（skills.shのインストール数で計測できる）
- 実測データ（日本語0件のスクショ）が引用・保存されるか
- 「エージェント型」という配布形態が伝わるか

## 2. issue-structuring — 2026-07-25

最初に「何を決めるか」を整える。対象者が広く、雑な相談や会議メモからすぐ試せるため、最初の体験を作りやすい。

確認する反応:

- 保存されるか
- 会議や資料作成前に使われるか
- Decision Askと「今やらない論点」が価値として伝わるか

## 3. deliverable-review — 2026-08-08

論点整理の次に、成果物を「意思決定に使える形」へ直す。誤字脱字ではなく、結論、根拠、依頼、構造をレビューする。

確認する反応:

- 資料レビュー需要が論点整理より強いか
- 「きれいなゴミ」という問題提起が共感されるか
- ビフォーアフターが保存・引用されるか

## 4. research-to-insight — 2026-08-22

最後に、より難度の高い「調べた事実を、示唆と行動へ変える」を出す。先に2本の利用例を作り、信頼が乗った状態で紹介する。

確認する反応:

- 要約ではなく示唆化への需要があるか
- 根拠、反証、確信度を含む出力が実務で使われるか
- 調査・企画業務の利用者が増えるか

## 更新方針

- 公開後2週間は新機能より利用例を集める
- 同じ不満が2件以上出たらv0.xを更新する
- Hooks、MCP、外部APIは共通コアから分離できる場合だけ追加する
- Claude CodeとCodexの両方で確認してから「動作確認済み」と表記する
